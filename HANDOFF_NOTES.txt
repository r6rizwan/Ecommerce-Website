PROJECT NAME: ShopSphere Ecommerce

SETUP INSTRUCTIONS:
- Run `npm install` in both `/backend` and `/frontend`
- Create `.env` files in `/backend` and `/frontend` using the respective `.env.example` files as reference
- Run the database schema file `schema.sql` before starting the project
- Run `npm start` in `/backend`
- Run `npm start` in `/frontend` for development, or serve the `/frontend/build` folder for production after running `npm run build`

DATABASE:
- Database name: `ecommerce`
- Schema file to run: `schema.sql`
- Tables created by schema:
  - `login`
  - `register`
  - `otp`
  - `category`
  - `product`
  - `feedback`
  - `customerOrders`
  - `payment`

CREDENTIALS NEEDED (to be filled by whoever is hosting):
- Backend `/backend/.env`
  - `PORT`
    - Backend server port
    - Can be set by hosting/server config
  - `ALLOWED_ORIGIN`
    - Frontend origin allowed by CORS
    - Must match the deployed frontend URL
  - `DB_HOST`
    - MySQL host
  - `DB_PORT`
    - MySQL port
  - `DB_NAME`
    - MySQL database name
  - `DB_USER`
    - MySQL username
  - `DB_PASSWORD`
    - MySQL password
  - `EMAIL_USER`
    - Nodemailer sender email
    - Needs a real email account
  - `EMAIL_PASS`
    - Nodemailer email password or app password
    - Needs a real email account/app password
  - `SUPER_ADMIN_EMAIL`
    - Super-admin login email
    - Should be created fresh for production
  - `SUPER_ADMIN_PASSWORD`
    - Super-admin login password
    - Should be created fresh for production
  - `SUPER_ADMIN_JWT_SECRET`
    - JWT secret for super-admin auth
    - Must be created fresh
  - `ADMIN_JWT_SECRET`
    - JWT secret for admin auth
    - Must be created fresh
  - `PAYMENT_PROVIDER`
    - Payment mode such as `razorpay` or `mock`
  - `RAZORPAY_KEY_ID`
    - Razorpay public key id
    - Needs a real Razorpay account for production
  - `RAZORPAY_KEY_SECRET`
    - Razorpay secret key
    - Needs a real Razorpay account for production
  - `RAZORPAY_MAX_AMOUNT_PAISE`
    - Maximum payment amount allowed by the backend, in paise

- Frontend `/frontend/.env`
  - `REACT_APP_API_URL`
    - Backend API base URL
    - Must point to the deployed backend
  - `REACT_APP_RAZORPAY_KEY_ID`
    - Razorpay Key ID used by the frontend checkout flow
    - Needs a real Razorpay account for production
  - `REACT_APP_PAYMENT_PROVIDER`
    - Frontend payment mode such as `razorpay` or `mock`

PORTS:
- Backend runs on: `PORT` from `/backend/.env` (recommended default: `5000`)
- Frontend runs on: React dev server default during development, or can be served from `/frontend/build` after `npm run build`
